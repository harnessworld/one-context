# Brief — one-context 中视频

## 成片参数

| 项 | 约定 |
|----|------|
| 工作标题 | 用 AI 写代码的人都踩过这些坑，这个开源项目一次全解决（可改） |
| 目标时长 | **3 分 20 秒（200s）** |
| 形式 | 双人问答口播 + 屏幕录屏 + 花字 |
| 平台（初定） | 抖音 / B站 / 视频号（可勾选） |

## 人设

| 角色 | 定位 |
|------|------|
| **老周** | 资深架构师，语速稳，给结论、讲机制 |
| **小林** | 一线开发，情绪外放，戳痛点、代观众发问 |

## 记忆点（可角标弱提示）

- **一处定义，处处同步**
- **共享上下文层**（不是「又一个 AI 模型」）

## 爆款结构（逻辑）

1. 前 3 秒炸场 → 10s 内三个具象痛点 → 30s 内抛出 one-context
2. 每约 15–20s 一个小高潮（命令、对比、反转）
3. 技术词当场大白话翻译
4. 互动埋点：评论区扣 1 / 扣 2（可按平台改）
5. 结尾：开源地址 + 关注理由（下期教程）

## 非目标（brief 层）

- 不讲实现源码细节（无行号 walkthrough）
- 不贬低具体厂商；强调「多工具、多仓库」结构性矛盾
