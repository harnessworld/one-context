# 话题大纲

## 1. 开场钩子 + 真实灾难（~1min）

- 悬念钩子："一个AI Agent，一条命令，删掉了整个生产环境"
- Alexey Grigorev 案例：terraform destroy 清空由 Terraform 管理的基础设施
- 194 万行数据瞬间消失
- 第二人称代入："想想你的生产环境——一个 Agent 一条命令就能全部清空，你受得了吗？"
- 核心论点：没有沙箱，AI Agent 就是定时炸弹

## 2. 三大认知误区（~1min，递进节奏）

- 误区一：沙箱只是安全工具 → 安全只是最基础功能
- 误区二：沙箱 = Docker 容器 → [预告案例] 下一个案例告诉你为什么不够
- 误区三：沙箱拖慢性能 → 预热池分配 60ms，比眨眼还快

## 3. 比喻：数字员工的专属办公室（~0.5min）

- Agent = 数字员工 → 沙箱 = 专属工作环境
- [闭环转场]"但如果办公室的墙是纸糊的，比没有还危险"

## 4. n8n 案例 + Docker 不够（~1min，从误区二自然过渡）

- n8n CVE-2024-29041（CVSS 9.8）Python 代码执行漏洞：应用层沙箱 ≠ OS 级隔离
- Docker 容器：共享内核 → 攻击面大 → 高风险场景需更强隔离
- [事实修正] CVE-2024-29041 证实应用层沙箱不够，容器共享内核风险需另述

## 5. 沙箱技术三层架构（~2.5min，比喻+数字精简版）

- 容器隔离 = 合租公寓共享大门｜gVisor = 前台代收快递｜microVM = 独立门禁独栋
- Firecracker 冷启动 ~125ms，预热池就绪 ~60ms，空沙箱 ~30MB
- Kata Containers 补充（OCI 兼容）
- [限定] microVM 高安全场景最具竞争力，但不是唯一选择
- Cube 开源（一句话带过，数据不重复展开）

## 6. Anthropic 三层解耦 + 场景化（~2min，用"Agent挂了怎么办"引入）

- 场景：Agent 执行到一半沙箱挂了 → 旧架构抢救 vs 新架构秒级恢复
- Session（append-only 日志）→ Harness（无状态调度）→ Sandbox（隔离执行）
- 核心洞察：沙箱从"宠物"变"牲口"，销毁重建即可
- [归因修正] 性能提升归因于架构解耦，非沙箱本身；数据来源 Anthropic 工程博客
- "从闭源到开源，从美国到中国" 收束

## 7. 三大核心价值——回马枪结构（~2.5min）

- 安全隔离：[回马枪] "还记得 n8n 那个漏洞吗？" → 凭证隔离原则 → [限定] 不解决 Agent 被操纵问题
- 可复现环境：销毁-重建（非暂停-恢复）→ 成本节省 50-80%（加限定条件）
- 弹性伸缩：[代入] "如果你公司跑着几百个 Agent，每个都能访问你的数据库密钥…" → 池化技术
