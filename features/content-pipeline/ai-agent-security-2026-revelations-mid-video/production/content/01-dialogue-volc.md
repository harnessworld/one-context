男：开发者朋友们，先问一句：你敢让 AI 编程助手在你没在盯着的时候，照样握着生产环境的执行权限吗？
女：别觉得是危言耸听——从 2025 年下半年到 2026 年春，公开复盘里已经串起一串真实事故：智能体对生产库执行迁移/同步命令却不带二次确认、Terraform 误指向生产状态文件后直接 destroy、`docker rm` 打掉带着业务数据的容器……多起案例上过技术社群热帖和开发者复盘文章。
男：英文圈里有人把这叫做同一种结构性翻车：智能体继承了操作员的过高权限，却对删除类、迁移类命令缺少强制确认闸——核心往往不是「模型偶然犯错」，而是我们把自动执行当成了省力，却没把不可逆操作当成高危入口。
男：（时间线可参考 Stack Futures《AI Coding Agents Keep Deleting Production》一文里的汇总表。）
女：Governance 从来不是什么大厂专属名词：只要你在用 Agent 写代码、跑工具链，你就已经在承担运维边界——差别只是你有没有把护栏装上。
男：【第一部分・四类典型翻车，本质都致命】
男：公开讨论里常见把 Agent 事故归成四大类，每一类都能让项目一夜回到解放前：
男：第一类，Prompt 注入与机密外泄。攻击者不要求你点恶意链接，也可能诱导模型把密钥、token、内网接口当成「任务的一部分」往外送。
女：第二类，长会话下的目标漂移。上下文堆高了以后，指令优先级容易糊掉——你以为还在测 dev，模型已经把变更推向_prod 路径；再加上工具默认参数偏向「帮你省事」，风险会被放大。
女：（具体窗口和行为因模型、IDE、插件而异，但机制是共通的。）
男：第三类，工具越权执行不可逆命令。复盘里反复出现的就是这一类：`terraform destroy`、强制数据库迁移、容器删除……共同点是没有对的人、在对的时机、按对的策略点下确认。
女：第四类，供应链与扩展生态。热门 MCP、Skill、插件并不等于做过威胁建模——装进来等于给 Agent 新开了一组 syscall / 网络面，恶意或粗心的组件一样能埋雷。
男：这四类的根只有一个：能力往前跑，治理没跟上。
男：【第二部分・为什么这两年格外刺眼】
女：很多人纳闷：以前聊 Copilot 好像没这么惊心？
男：一句话：Agent 从「给建议」变成了「能替你执行」——权限半径大一截，爆炸半径也大一圈。再加上营销演示里，「流畅一键」永远在 C 位，「审批、沙箱、最小权限」常常被一笔带过。
女：不是厂商不懂安全，而是「严格的门禁」在短片里不吸睛，受众也更愿意为「省时间」买单——结果迭代越快，欠账越多。
男：这和当年 OpenSSL 心脏滴血、Log4j 远程执行有点像：生态先狂奔，事故倒逼补课——只是这一次，背锅位离独立开发者更近。
女：Agent 不是天降特例，是软件安全的老规律换了一条跑道而已。
男：【第三部分・护栏已经在开源里长出来了】
男：好消息是：用不着先等「终极模型」——运行时治理、策略拦截、审计链路，社区和厂商都在铺。
女：举四个能说清定位的例子，细节以各自 README 为准：Rust 方向的 Agent Armor，强调执行前策略管线、审批与观测；
女：微软开源的 Agent Governance Toolkit，仓库里自带与 OWASP Agentic AI 风险项的对照文档；
女：GitHub 上的 Governor，面向长会话做上下文压缩和漂移护栏；
女：Cordum 则给 LangChain、CrewAI 等栈提供策略评估与审计轨迹——本质都是「先拦截、再执行、可追责」。
男：独立研究者 Jason Zhu（X：@GoSailGlobal）也在维护 Agent Skills 方向的索引与讨论——装扩展前先看来源、看出处、看 issue，和他做的事其实是同一条防线：供应链也要当代码审。
女：我们不在这里背星星数——你去点开许可证、发布频率和安全讨论，比听我念一个总数靠谱。
男：治理从来不是「买最贵的盒子」，而是把对的钩子挂在对的执行路径上。
男：【第四部分・三件事，今天就能做】
女：工具再多，也得落地。先把这三条养成肌肉记忆：
男：第一件，给高危操作加确认门。删文件、跑 Shell、碰生产数据库、跑 IaC destroy——默认视为红色命令，必须二次确认或拆成「仅预览计划」步骤。
女：第二件，给上下文设预算。长对话必压缩或分段重置；别让模型在噪音里自己改写自己的目标——预算一到就先归档小结，再开新线程。
男：第三件，装链路先看供应链。作者是谁、多久更新、issue 里有没有人报权限过大或可疑行为——高 star、高下载都不等于安全背书。
男：【结尾・收束】
女：今天聊的不是否定 AI，而是让 AI 在边界里交付价值。
男：模型会越来越强，但「谁能执行、执行到哪、谁能撤回」仍是人的责任——公开复盘已经告诉我们：缺的不是段子，是门禁。
女：别等上热搜才补课。治理不是职务头衔，是每个把 Agent 接进工作流的人要扛的日常。
男：你在长会话里遇到过跑偏、误执行吗？评论区聊聊；觉得有用就转给还在「一键全开权限」的同事。
女：用好 AI，先把笼子焊牢——先做安全的开发者，再做高效的开发者。
