共同机制

智能体继承了操作员的过高权限；对删除类、迁移类缺少「对的人 × 对的时机 × 对的策略」三重对齐。

治理不是大厂专利

只要 Agent 写代码、跑工具链，你就已在承担运维边界。

第一部分 · 四类典型风险

分类

每一类都能一夜清零

① 注入与泄密

密钥 / token / 内网接口被诱导外送。

② 目标漂移

长会话堆上下文；工具默认替你「省事」。

③ 越权不可逆

destroy / 迁移 / 删容器——缺确认闸。

④ 供应链

新 syscall、网络面随扩展进来。

四类的根只有一个：能力往前跑，治理没跟上。

第二部分 · 为何这两年格外刺眼

范式迁移

从「建议」到「执行」

参谋只动嘴 → 副驾握方向盘：权限与爆炸半径同步放大。

迭代越快，欠账越多——和 OpenSSL / Log4j 同理，只是跑道换成 Agent。

第三部分 · 护栏已在开源生长

矩阵

不必等「终极模型」

拦截 → 执行 → 观测；可参考 OWASP Agentic 等公开清单做映射。

治理是把钩子挂在对的执行路径上，不是买最贵的盒子。

第三部分 · 扩展面

供应链

装插件 = 新开 syscall / 网络面

装之前当代码审

作者、更新频率、issue 里权限争议——热门 MCP / Skill ≠ 威胁建模过。

索引与讨论

社区里有 Agent Skills / MCP 等资源索引；装扩展前看出处、读 issue，与「审供应链」是同一件事。

别只看 star——许可证与安全讨论更值得点开。

第四部分 · 今天就能做

落地

三件事养成肌肉记忆

① 高危默认红色 ② 上下文预算 ③ 供应链审查

删文件 / Shell / 生产 DB / IaC destroy → 二次确认或「仅预览」。

长对话压缩或分段；预算一到先归档小结再开新线程。

装链路前查作者、更新、可疑行为——高 star 不是背书。

肌肉记忆比囤工具清单管用。

结尾 · 不是否定 AI

态度

老规律，新跑道

生态先狂奔
心脏滴血 · Log4j 同款叙事

事故倒逼补课——这次责任主体离一线开发者更近。

让 AI 在边界里交付价值。

谁能执行 · 执行到哪 · 谁能撤回——仍是人的责任。

先做安全的开发者，再做高效的开发者。
