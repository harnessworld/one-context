05 / 08
Counter-intuitive Rules

三大陷阱

📨
不要改 System Prompt
变化信息通过 user message 传递，不动前缀。
Claude Code 用 <system-reminder> 注入更新
🚫
不要换模型
缓存模型绑定。100k Opus 换 Haiku 反而全价重算。
用 subagent fork 独立缓存路径
🔧
不要换工具集
加减工具 = 前缀失效。用 defer_loading stubs 替代移除。
Plan Mode 是工具，不是换工具集
用 message 更新，用工具建模状态，用 subagent 切换路径。
维系前缀 = 维系成本
前缀不变，成本永低

07 / 08
Design Principles

Architecture as Caching

🎯 顺序 = 架构
前缀匹配约束决定系统设计，顺序不是细节。
📨 Message 传变化
所有变动通过 message 追加，不动 system prompt。
🔧 工具建状态
Plan Mode / defer_loading 是工具行为，非工具集变更。
📊 命中率 = Uptime
缓存命中率与在线率同级，波动即告警。
Claude Code 从 day one 就以 caching 为架构基础。
