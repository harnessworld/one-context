## 简介

当 Agent 从「偶尔调用」变成「自生长子任务、黑盒决策、跨服务乱窜」，你的集群赌的是：**弹性跟不跟得上、GPU 空不空转、工具慢不慢、状态散不散、安全守不守得住**——输一次就是成本与事故，不是多开几个 Pod 能补的。

这期用一张主线的「障碍 → 亲和」图，把 **推理/GPU 亲和、动态组批、工具与网络微秒路径、分层状态、全链路安全隔离** 串成一条可执行的架构叙事：不是概念堆砌，而是**你在生产里必须押的那几手**。

**适合谁听**：已经上或准备上 Agent / LLM 服务的工程与架构同学；想从「能跑 demo」走到「敢上流量」的人。**看完后你该带走**：传统云的四类硬伤、四类亲和分别在赌什么取舍、以及收口时如何一眼对齐团队话术。
