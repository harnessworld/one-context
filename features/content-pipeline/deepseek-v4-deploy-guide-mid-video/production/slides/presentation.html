对接时的四条线

Integration

  • 思维链：与消息历史一致的回传策略。
  • 流式：首包、网关超时与背压。
  • 前缀缓存：命中失败会直接打在延迟与成本上。
  • 并发灰度：最后才谈 SLA；顺序错了全歪。

四类常见症状

多轮漂移

统一 assistant / reasoning 字段回传；别让客户端随机丢段。

首包慢

stream、网关超时、前缀缓存三连；先量瓶颈再调参。

排队爆延迟

并发与引擎队列对齐；客户端退避别「打穿」网关。

特性开关 + 版本 pin；预留降级路径，别绑死在单一路由。

部署

推理栈怎么选

Runtime

  • vLLM：生产默认解；PagedAttention + MoE 生态成熟。
  • Ollama / SGLang：轻量原型或低延迟专用场景评估。
  • 换栈：权重、量化与内核制品混用是高风险源。

先锁定制品，再谈 QPS。

部署07
我们是强烈推荐就是用VLLM
硬件

容量一条线

Footprint

  • 算力链条：全精度 → 量化 → Lite，逐级减卡。
  • 多卡场景优先捋清 NVLink / 拓扑，别只看显存容量。
  • 内存、本地盘与 OS/CUDA 版本写在清单里，避免「机器够却跑不稳」。

硬件表述随机型调整；结构不变。

四条防线

Ops — 文本是 checklist，图标减轻认知负荷

制品同源

权重、tokenizer、推理镜像同一 CI；禁止「手搓目录」。

上下文上限

对齐服务端 max_len；静默截断最难排查。

显存开关

Prefix cache、共享内存、内核特性要与栈版本匹配。

交付弹性

容器探针、滚动发布、入口 Router 与熔断要写进 Runbook。
