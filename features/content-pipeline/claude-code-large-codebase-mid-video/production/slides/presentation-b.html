LSP · Precise Navigation

第五层：给 Claude 和 IDE 一样的导航

没有 LSP，在百万行代码库里 grep 一个 handle 或 process → 几千个结果，逐个打开判断 → context 烧完。

LSP 提供跳转到定义、查找所有引用——按符号精确导航，区分不同语言里同名函数的差别。

企业案例

一家企业软件公司在全组织部署 Claude Code 之前，先铺了 LSP 集成——专门为 C/C++ 代码导航更可靠。

第五层是 LSP，把 IDE 的精确导航能力暴露给 Claude

10 / 15
Sub-Agent · Independent Context

第七层：探索与编辑分离

子 Agent 是独立的 Claude 实例，有自己的 context window。做完任务只把最终结果返回给主 Agent。

关键洞察：在同一个会话里又探索又编辑，context 很容易被撑爆。子 Agent 的探索过程被限制在自己的 context window 中，不会污染主 Agent。 Explore first, edit later
实用模式

1. 只读子 Agent 摸清子系统 → 写成文件
2. 主 Agent 带完整上下文做编辑

第七层是子 Agent，独立的 Claude 实例，有自己独立的 context window
