# 口播讲稿（主终稿）

**主播**：阿哲（技术视角，理性务实）、小夏（行业观察，敏锐通透）  
**风格**：对话自然、有干货、不哗众取宠、引用权威须已核查（见 `spec.md` 开放问题）。

---

## 开场（抓耳不浮夸）

**小夏**：最近技术圈真的炸了！一个来自 Claude Code 团队的工程师 Thariq，直接在 X 平台公开说：我已经彻底不用 Markdown 了。

**阿哲**：没错！这条帖子几小时就破 200 万浏览，直接引爆争议 —— 陪伴程序员十几年的 Markdown，真的要被 HTML 取代了吗？

**小夏**：今天咱们不跟风吵，只聊大佬实测、真实痛点、行业本质，听完你就知道该用什么、怎么选。

---

## 一、先认账：Markdown 曾经有多神

**阿哲**：先给 Markdown 一个公道。它能统治技术圈这么久，真的靠硬实力：纯文本、极简语法，随手就能写；版本控制友好，GitHub、笔记软件全标配；几乎全平台兼容，是轻量标记语言的代名词。

**小夏**：我懂，就是 「够用、好写、不折腾」。但为什么 Claude Code 整个团队，都开始转向 HTML 了？

---

## 二、灵魂拷问：为什么大佬主动放弃 Markdown？

**阿哲**：Thariq 原话很扎心：我都不亲自编辑文件了，基本靠 Claude 改，Markdown 最核心的 「手动易编辑」优势，直接被废掉了。

**小夏**：再加上 AI Agent 越来越强，Markdown 的短板彻底暴露：长文档死刑：超过 100 行，自己都读不下去；信息密度太低：复杂表格、颜色、图表根本展不开；分享极麻烦：浏览器不原生支持，只能发附件。

**阿哲**：简单说：AI 时代，Markdown 撑不住复杂协作了。

---

## 三、HTML 凭什么碾压 Markdown？大佬亲测 4 大优势

**小夏**：Thariq 直接给 HTML 盖戳：Claude 能读懂的信息，HTML 几乎都能高效呈现。到底强在哪？

**阿哲**：4 个核心优势，全是实战痛点：表达力拉满：表格、插图、样式、图表 8 类信息随便放，告别 ASCII 丑图；阅读体验翻倍：标签导航、响应式布局，像看网页一样舒服；分享零成本：上传就出链接，不用发附件，打开率高太多；双向交互：这是王炸！能加滑块、按钮、控制器，调参数直接同步回 Claude。

**小夏**：原来不是好看，是能交互、能闭环，完全适配 AI 协作。

---

## 四、为什么只用 Claude Code 生成 HTML？

**阿哲**：很多人问：为啥不用 Claude AI 或 Claude Design？Thariq 的答案很专业：Claude Code 的上下文能力碾压—— 能读当前文件、扫代码库、查 git 历史、连 MCP 工具（Slack 等），直接基于全量信息生成页面。

**小夏**：相当于 AI 带着 「全记忆」干活，不是凭空写。

---

## 五、落地干货：工程师私藏 5 大 HTML 使用场景

**阿哲**：光说不练假把式，Thariq 直接给了 5 个能直接抄的场景：发散思考：AI 生成多版 HTML 页面，网状探索，不是单线程写稿；代码审查：HTML 渲染差异、彩色注释，比 GitHub 原生视图还好用，PR 直接附 HTML 解释器；设计原型：快速出高保真交互，再转 React/Swift，效率起飞；专业报告：整合多数据源，生成可读可交互的报告 / 演示稿；自定义编辑器：为单一任务做一次性 HTML 界面，精准解决问题。

**小夏**：每一个都是提升效率的真刚需。

---

## 六、灵魂三问：耗 Token、慢、版本差怎么办？

**小夏**：质疑声肯定有，Thariq 也坦诚回应了，很客观：

**阿哲**：Token 更费？HTML 表达力更强，整体输出更好，在 Opus 4.7 百万字上下文里，增量可以忽略。生成更慢？确实慢 2-4 倍，但结果值得。版本控制差？这是 HTML 最大短板，对比差异太 「吵」，这一点承认不如 Markdown。

**小夏**：不吹不黑，这才是理性选型。

---

## 七、深度洞见：工具不会死，只会找对生态位

**阿哲**：最后聊核心 ——Markdown 会被淘汰吗？Excalidraw 会消失吗？

**小夏**：答案是：不会。Thariq 自己都认可：每一种工具都有专属生态位。Markdown：极简、版本友好，快速笔记、简单文档依然无敌。Excalidraw：手绘白板，快速草稿、头脑风暴无可替代。HTML：AI 协作、复杂表达、交互闭环，成为新的最优解。

**阿哲**：本质不是 「谁淘汰谁」，而是 AI 时代，工具要适配 「人 + AI」的新协作方式。

**小夏**：以前是 「人写、人看」，现在是 「AI 生成、人决策、双向交互」——HTML 刚好补上这个缺口。

---

## 结尾（收束有价值）

**阿哲**：总结一句话：Markdown 没有死，只是回到它最擅长的位置；HTML 崛起，是因为适配了 AI 协作的新需求。

**小夏**：不用跟风换工具，先看你的场景：简单笔记用 Markdown，AI 协作、复杂表达、交互需求，直接试 Claude Code+HTML。

**阿哲**：工具永远服务于人，选对场景，比追风口重要一万倍。
