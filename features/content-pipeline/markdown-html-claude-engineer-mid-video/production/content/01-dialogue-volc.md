# 火山播客 TTS 对白稿（action=3）

**映射**：`男：` = **阿哲**，`女：` = **小夏**（与 `skills/volc-podcast-tts` 默认男女声一致时可对调控制台发音人）。

---

女：最近技术圈真的炸了！一个来自 Claude Code 团队的工程师 Thariq，直接在 X 平台公开说：我已经彻底不用 Markdown 了。

男：没错！这条帖子几小时就破 200 万浏览，直接引爆争议 —— 陪伴程序员十几年的 Markdown，真的要被 HTML 取代了吗？

女：今天咱们不跟风吵，只聊大佬实测、真实痛点、行业本质，听完你就知道该用什么、怎么选。

男：先给 Markdown 一个公道。它能统治技术圈这么久，真的靠硬实力：纯文本、极简语法，随手就能写；版本控制友好，GitHub、笔记软件全标配；几乎全平台兼容，是轻量标记语言的代名词。

女：我懂，就是够用、好写、不折腾。但为什么 Claude Code 整个团队，都开始转向 HTML 了？

男：Thariq 原话很扎心：我都不亲自编辑文件了，基本靠 Claude 改，Markdown 最核心的手动易编辑优势，直接被废掉了。

女：再加上 AI Agent 越来越强，Markdown 的短板彻底暴露：长文档超过一百行，自己都读不下去；信息密度太低，复杂表格、颜色、图表根本展不开；分享还麻烦，浏览器不原生支持，只能发附件。

男：简单说：AI 时代，Markdown 撑不住复杂协作了。

女：Thariq 直接给 HTML 盖戳：Claude 能读懂的信息，HTML 几乎都能高效呈现。到底强在哪？

男：四个核心优势，全是实战痛点：表达力拉满，表格、插图、样式、图表随便放，告别 ASCII 丑图；阅读体验翻倍，标签导航、响应式布局，像看网页一样舒服；分享零成本，上传就出链接，不用发附件；双向交互是王炸，能加滑块、按钮、控制器，调参数还能同步回 Claude。

女：原来不是好看，是能交互、能闭环，完全适配 AI 协作。

男：很多人问：为啥不用 Claude AI 或 Claude Design？Thariq 的答案很专业：Claude Code 上下文能力更强，能读当前文件、扫代码库、查 git 历史、连 MCP 工具，直接基于全量信息生成页面。

女：相当于 AI 带着全记忆干活，不是凭空写。

男：光说不练假把式，Thariq 给了五个能直接抄的场景：发散思考，多版 HTML 网状探索；代码审查，HTML 里做渲染差异和彩色注释，PR 附解释器；设计原型，高保真交互再转框架；专业报告，多数据源可读可交互；自定义编辑器，一次性界面精准解决问题。

女：每一个听起来都是真刚需。

女：那质疑呢？Token、速度、版本管理怎么办？

男：Thariq 回应得很客观：HTML 表达力更强，在超大上下文模型里 Token 增量往往可以接受；生成确实可能慢上几倍，但结果值得；版本 diff 噪音大，这点 HTML 不如 Markdown，得认。

女：不吹不黑，这才是理性选型。

男：最后聊核心：Markdown 会不会死？Excalidraw 会不会消失？

女：不会。Thariq 自己也说，每种工具都有生态位。Markdown 管极简笔记和简单文档；Excalidraw 管手绘白板头脑风暴；HTML 管 AI 协作、复杂表达和交互闭环。

男：本质不是谁淘汰谁，而是协作从人写人看，变成 AI 生成、人决策、还能双向交互。

女：总结一句：Markdown 回到擅长的位置，HTML 补上 AI 协作的新缺口。

男：别跟风换栈，先看场景：简单笔记 Markdown；要强表达、交互和协作闭环，试试 Claude Code 加 HTML。

女：工具服务于人，选对场景，比追风口重要一万倍。
