## P0 问题（7 个，全部已解决）

### P0-1 开场钩子力度不足

| 维度     | 说明 |
| -------- | ---- |
| 提出角色 | 短视频用户、短视频专家 |
| 问题现象 | 开场「你做的每一份汇报都是在做一个短视频」概念感太强，情绪唤起不够 |
| 问题成因 | 用了观点型钩子而非痛感型钩子，用户第一反应是"又来教我做事" |
| 问题影响 | 前 5 秒留存率预估不超过 35%，直接影响完播率 |
| 定级依据 | 开场是完播率的决定性因素，失去前 5 秒就失去整个视频 |

**解决方案**：改为具体场景 + 反常识结果——「上周汇报，老板20页PPT只看了3页——你猜他批了没有？」「批了。」

**修改前**：
> A: 你做的每一份汇报，都是在做一个短视频。B:（愣了一下）……啊？

**修改后**：
> B: 上周汇报，老板20页PPT只看了3页——你猜他批了没有？A:（停顿一秒）批了。B: ……你怎么知道的？

### P0-2 4分钟时长在抖音是死亡区间

| 维度     | 说明 |
| -------- | ---- |
| 提出角色 | 短视频用户、短视频专家 |
| 问题现象 | 4 分钟定位为三平台通用 |
| 问题成因 | 抖音完播率算法对 4 分钟内容要求极高，知识类甜区是 1:30-2:30 |
| 问题影响 | 抖音完播率预估仅 8-12%，难过自然流量池 |
| 定级依据 | 平台适配直接决定内容能否被推荐 |

**解决方案**：在结构方案中增加抖音 2 分钟精简版裁剪方案，明确三平台分发策略。

**修改**：在 `00-structure.md` 中新增「抖音 2 分钟版裁剪方案」和「封面方案」。

### P0-3 原则拆解段 50 秒纯干货是"死段"

| 维度     | 说明 |
| -------- | ---- |
| 提出角色 | 短视频用户、短视频专家 |
| 问题现象 | 1:10-2:00 共 50 秒纯讲 4 条原则，条列式递进 |
| 问题成因 | 知识类视频的完播杀手，连续说教触发"上课感" |
| 问题影响 | 用户在第 2 条就会走神，中段流失严重 |
| 定级依据 | 中段流失会导致完播率断崖，算法降权 |

**解决方案**：原则拆解与 Before/After 示范融合，边讲原则边演绎场景，B 每条提出质疑增加对谈感。

**修改**：将原来独立的「原则拆解」和「场景示范」合并为「原则拆解 + 场景示范（融合）」。

### P0-4 原则③"信息密度递减"与职场实际矛盾

| 维度     | 说明 |
| -------- | ---- |
| 提出角色 | 职场人、职场老板 |
| 问题现象 | 「信息密度递减」意味着后半段变稀疏，老板追问时接不住 |
| 问题成因 | 短视频的信息密度模型不能直接套用到需要深度交互的汇报场景 |
| 问题影响 | 观众照做后被老板追问细节时露馅，降低方案可信度 |
| 定级依据 | 核心原则有误导性，会让目标受众觉得"这人没在真实职场待过" |

**解决方案**：改为「先结论后论证，细节按需展开」——前端高密度给结论，后端准备好细节弹药但不主动倾泻。

**修改前**：
> ③ 信息密度递减。先给结论，再给论证。

**修改后**：
> ③ 先结论后论证，细节按需展开。

### P0-5 原则④"情绪锚点取代数据堆砌"有误导风险

| 维度     | 说明 |
| -------- | ---- |
| 提出角色 | 职场人、职场老板 |
| 问题现象 | 措辞暗示"不要数据"，与职场数据驱动文化冲突 |
| 问题成因 | "取代"一词过于绝对，老板需要数据做决策 |
| 问题影响 | 在数据驱动公司照做会被打回，降低方案可信度 |
| 定级依据 | 核心原则表述可能导致实际汇报翻车 |

**解决方案**：改为「数据是弹药，情绪锚点是瞄准器——先让对方在乎，再给数据」。

**修改前**：
> ④ 情绪锚点取代数据堆砌。3个数字不如1个画面感。

**修改后**：
> ④ 数据是弹药，情绪锚点是瞄准器。先让对方在乎，再给数据。数据和画面感是配合关系，不是替代。

### P0-6 "您选哪个"二选一逼决策

| 维度     | 说明 |
| -------- | ---- |
| 提出角色 | 职场人、职场老板 |
| 问题现象 | 要资源 After 版「差2个人，有6周没有10周，您选哪个」预设二选一 |
| 问题成因 | 限制了老板的决策空间（砍需求、借人、分期交付等选项被排除） |
| 问题影响 | 经验丰富的管理者会觉得被"框"了，产生反感 |
| 定级依据 | 核心示范如果翻车，会动摇整个方案的说服力 |

**解决方案**：给出三条路径（加人/延期/砍需求），把决策空间留给老板。

**修改前**：
> 差2个人。有，6周上线；没有，10周起步。您选哪个？

**修改后**：
> 按当前资源预计10周。如果补2个人可以压到6周。如果人力和周期都动不了，我们可以聊聊哪些需求后置。
