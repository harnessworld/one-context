# 概述

短视频的注意力设计（前 3 秒钩子、15 秒小高潮、信息密度递减、情绪锚点）本质上是一种高效的信息传递范式——它让大多数人能集中注意力、能听进去逻辑。将这套范式迁移到职场汇报场景：**把老板当做短视频的用户来服务**，用内容创作的思路打造汇报范式。

这不是做视频工具，而是将短视频的「用户思维」迁移到汇报设计——让你不擅长 PPT 也没关系，因为你不需要做 PPT，你需要做的是「内容」。

# 目标与非目标

## 目标

- 提炼短视频注意力设计的核心原则体系：钩子结构、叙事节奏、信息层级、情绪锚点。
- 建立从短视频原则到汇报场景的映射关系（如：前 3 秒钩子 → 汇报开场的冲击陈述；信息密度递减 → 先结论后论证）。映射的核心前提：**在场是被迫的，注意力不是**——听众的脑子随时可以「滑走」，与刷短视频无本质区别，因此吸引注意力的诉求完全一致。
- 输出可复用的汇报范式模板：结构模板（开场/主体/收尾的衰减曲线）+ 节奏模板（何时设小高潮、何时留白）+ 话术模板（钩子句式、转折句式、收尾句式）。
- 提供至少 3 个具体场景的范式应用示例：周报、项目汇报、资源申请。

## 非目标

- 不做视频生成工具或自动化汇报工具。
- 不做 PPT 模板美化——范式与工具无关，口头汇报同样适用。
- 不承诺适用于所有汇报类型（合规/审计类汇报有固定格式要求，不纳入范式覆盖范围）。

# 用户与场景

- **技术人做周报/月报/项目汇报**：不擅长 PPT，但逻辑清晰，需要一种让逻辑被「听到」的结构。
- **产品人做需求评审**：需要在有限时间内让利益相关方听懂并认同。
- **任何人向上沟通、争取资源、汇报成果**：核心需求是——让对方愿意听、听得进去、记得住。
