# 口播讲稿

（双主播对话，轻松有深度，适配播客/短视频口播）

## 开场（抓耳不浮夸）

**A**：你敢相信吗？一位写过 TypeScript 教材、造过 Claude Code 的顶尖工程师，**2026 年至今一行代码都没写过**。

**B**：不止这个，他还在一天之内，提交了**150 个 PR**。

**A**：这不是科幻，是 Anthropic 工程负责人 Boris，在红杉现场亲口说的真相。

**B**：今天咱们不聊工具测评，只聊一件事：**工程师的真正变革，才刚刚开始**。

---

## 第一问：AI 编程，早就不是按 Tab 补一行代码了

**A**：我第一反应是，他不写代码，那活儿谁干？

**B**：模型干。而且不是补全，是**直接写完整代码**。

**A**：Claude Code 一开始就这么神吗？

**B**：完全相反。前 6 个月几乎不好用，他自己也只用它写 10% 的代码。

**A**：那为什么不放弃？

**B**：因为他们看到了关键——**产品先于模型成熟**。模型能力在狂奔，市面上的产品，根本没接住这份能力。

**B**：你记住这句话：**早期难用≠方向错了**。只要模型曲线能追上，今天的短板，几个月后就会被抹平。

---

## 第二问：未来的工程师，在手机上管几百个 Agent

**A**：不写代码，他每天干嘛？

**B**：用手机，管 Agent。他手机上同时开 5–10 个会话，每个会话里一堆 Agent，**总共几百个在并行跑**；晚上还能跑几千个做深度工作。

**A**：一天 150 个 PR，这效率太夸张了。

**B**：这就是**Agent 化开发**的本质：人从「敲每一行代码」，变成「组织一大批 Agent 干活」。

**A**：那最核心的工具是什么？

**B**：**Loop**，他说这就是未来。用定时任务，让 Agent 在后台自动修 CI、自动 rebase、处理不稳定测试、聚合用户反馈。

**B**：以前这些事靠人盯，现在 Agent 就是 24 小时在岗的后台同事。**改变工作密度的，不是一次性生成代码，是持续循环的自动化**。

---

## 第三问：你的团队，很快会人人写代码

**A**：这么说，普通工程师要被替代了？

**B**：恰恰相反，**会写代码的人会变多，但不是都变成传统程序员**。

**A**：什么意思？

**B**：Boris 的团队里，**工程经理、产品、设计、数据、财务、用户研究，全都写代码**。

**A**：财务也写？

**B**：对。代码不再是专业壁垒，而是**表达想法的工具**。产品直接改原型，设计直接做交互，业务直接生成小工具，不用等工程排期。

**B**：未来的团队，会长出大量**跨学科通才**，专长还在，但把想法变成可运行软件的门槛，直接被打掉了。

---

## 第四问：软件行业，迎来了「印刷机时刻」

**A**：这个变化，到底有多大？

**B**：Boris 用了一个绝 analogy——**15 世纪的印刷机**。

**A**：怎么讲？

**B**：印刷机之前，欧洲只有 10% 的人识字，读写是职业；印刷机之后 50 年，出版文献超之前 1000 年，成本降 100 倍。

**B**：软件也一样。以前写代码是少数人的专业，未来会像**发短信一样普遍**，比 Office 还要普及。

**A**：那软件会变少吗？

**B**：不会。会**更多、更碎、更贴近业务**。以前不值得排期的小工具，以后人人都能做。工程师的稀缺性，从「会不会写」，变成**能不能定义问题、组织 Agent、定验证标准**。

---

## 第五问：真正的护城河，不是工具，是流程

**A**：那大公司和小公司，谁更占优？

**B**：**新公司赢面更大**。

**A**：为什么？大公司不是资源更多吗？

**B**：大公司要改的不只是工具，是**流程、协作方式、全员习惯**，还有内部阻力。新公司可以从第一天，就按 AI 原生搭建组织。

**A**：所以技术都能买到，差距在哪？

**B**：在**谁能把工作拆成 Agent 能处理的任务**。还把 AI 当单人提效插件的团队，会被用多 Agent 协作的团队，彻底拉开速度差。

**B**：还有 MCP，打通 Salesforce、Google Docs 这些云工具，让知识工作和 AI 无缝衔接，模型自己选本地还是云端，人只需要说清目标。

---

## 结尾洞见（扎心又有用）

**A**：听完这些，我最大的感受是：不是 AI 取代人，是**用 AI 的人取代不用 AI 的人**。

**B**：对。Boris 的话很实在：**别纠结工程师会不会消失，先拆自己的工作**。

**A**：能不能变成 Agent 可执行、可验证、可循环的任务？

**B**：今天先改一条 CI、一个反馈流、一个定时报告，你就已经站在了下一轮软件生产的前面。

**B**：未来的核心竞争力，从来不是敲多快的代码，而是**你有多会指挥 AI，把想法变成价值**。
