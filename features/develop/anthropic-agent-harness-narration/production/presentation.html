CORE LOOP
Agentic Loop 🔄
与一次性推理的本质区别 — 就像牧羊犬不停巡回 🐕
🐕 牧羊犬巡回
对比
一次推理 vs 循环推理
❌ 一次推理 One-Shot
🐱 单次，无反馈
🐱 无法纠偏，一锤子买卖
🐱 遇错即崩，没有恢复机制
✅ Agentic Loop
🐕 多轮反馈，自动纠偏
🐕 持续迭代，越跑越好
🐕 遇错修复，真正的智能体
💡 关键差异
Loop 让 Agent 能观察执行结果，基于反馈调整下一步。缺少循环 = 不是真正的智能体 🐾→🐕
Anthropic 说"LLM 不是 Agent"，因为没有循环和工具，它只是一次性推理引擎。

容错 🐑
挂了也不怕
任何一个组件挂了，都不影响整个系统
Harness 挂了
新实例无缝接管
会话状态在 Session 层
生态继续运转 🐕‍🦺
Sandbox 挂了
自动重启
错误隔离在沙箱内
不会扩散 🐑
🛡️ 核心容错机制
• 会话状态保存在外部，不随实例消亡
• 新实例恢复现场后继续执行，用户无感
• 任何一个组件挂掉，其他层不受影响
• 很难出现整个系统级崩溃 🐖
