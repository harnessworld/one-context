### 事实澄清（勿误说 MCP）

**one-context 不靠 MCP 协议做多端适配。** MCP 是 IDE/客户端里「模型 ↔ 外部工具」的**运行时**协议；本项目是 **本地 CLI**（`onecxt`）读取 `meta/`、`knowledge/` 等**权威清单**，经**适配器**把同一套语义**写成** Cursor / Claude Code / OpenClaw 等各自的**配置文件**（如 `.mdc`、适配器 Markdown、JSON）。这是**文件生成与同步**，与是否接入 MCP **无关**。口播、字幕、简介里请**不要**写「通过 MCP 实现」。
